Detailed Description

Container for a block tridiagonal linear system.

The matrix has the structure

\[\begin{bmatrix} D_0 & U_0 \\ L_0 & D_1 & U_1 \\ & L_1 & D_2 & \ddots \\ & & \ddots & \ddots & U_{m-2} \\ & & & L_{m-2} & D_{m-1} \end{bmatrix} \]

where each block has size \(n \times n\).

The implementation stores separately:

  • the diagonal blocks D_bloks,
  • the lower off-diagonal blocks L_bloks,
  • the upper off-diagonal blocks U_bloks,
  • the right-hand side and the solution.

The system is solved with a block Thomas algorithm, interpreted as an LDU factorization.

Allocate the data structure.

Allocate and zero the block tridiagonal structure.

Parameters
nsize of each square block.
n_blknumber of diagonal blocks.

After setup, all arrays are reset to zero.

Memory is organized in "stripes":

  • diagonal blocks are placed side by side in D_bloks,
  • the same idea is used for lower and upper off-diagonal blocks.

This makes both pointwise access and extraction of whole blocks through middleCols straightforward.

◆ solve()

void Exercise2::Block_Tridiagonal::solve ( )

Solve the linear system with the block Thomas algorithm.

Solve the system with block Thomas ⇔ LDU factorization.

The procedure performs:

  • forward elimination on the modified diagonal blocks;
  • local dense solves on the blocks;
  • backward substitution.
Exceptions
std::runtime_errorif the structure is not initialized or if one of the modified diagonal blocks is singular.

Numerical scheme:

  • step 1: recursive construction of the modified diagonal blocks;
  • step 2: solve for the modified upper blocks Û_i;
  • step 3: forward elimination on the right-hand side;
  • step 4: backward substitution on the solution.

No explicit block inverse is ever formed: each occurrence of D^{-1} is implemented as a local dense solve.

FullPivLU is robust and simple for small dense blocks.
